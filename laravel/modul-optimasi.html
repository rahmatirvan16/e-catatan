Optimasi & Otomatisasi

Pelajari cara mengoptimalkan aplikasi Laravel menggunakan Redis untuk caching, mengelola background jobs dengan Laravel Horizon, dan memastikan kualitas kode dengan Pest/PHPUnit.

Pengenalan Redis

Redis (Remote Dictionary Server) adalah in-memory data store yang sangat cepat. Di Laravel, Redis digunakan untuk:

KegunaanManfaat
CacheSimpan hasil query DB yang lambat, response tidak berubah sering
SessionSimpan session user lebih cepat daripada database/file
QueueAntrian background jobs (email, notifikasi, export)
Rate LimitingBatasi jumlah request per user/IP

Redis vs Database Cache

AspekDatabase CacheRedis Cache
Kecepatan~10ms<1ms
StorageDisk (HDD/SSD)RAM
Cocok untukData yang jarang diaksesData yang sering diakses
SetupTidak perlu install tambahanPerlu install Redis server
Analogi
Redis seperti meja kerja — kamu menaruh dokumen yang sering dipakai di meja agar mudah dijangkau, bukan harus bolak-balik ke lemari arsip (database).

Instalasi Redis

Langkah 1 — Install Redis Server

Windows (via Memurai atau WSL):

Terminal (WSL/Ubuntu)
sudo apt update
sudo apt install redis-server
sudo service redis-server start
redis-cli ping   # harus balas: PONG

Alternatif Windows: Memurai — download dari memurai.com, install, langsung bisa dipakai tanpa WSL.

Langkah 2 — Install Predis (PHP Client)

Terminal
composer require predis/predis

Langkah 3 — Konfigurasi .env

.env
REDIS_HOST=127.0.0.1
REDIS_PASSWORD=null
REDIS_PORT=6379

CACHE_STORE=redis
SESSION_DRIVER=redis

Langkah 4 — Verifikasi Koneksi

Terminal
php artisan tinker
>>> Redis::ping();   // output: "+PONG"

Cache dengan Redis

Laravel menyediakan facade Cache yang bekerja sama di semua driver — termasuk Redis. Ganti driver di .env tanpa ubah kode aplikasi.

Operasi Dasar Cache

app/Http/Controllers/Api/ProdukController.php
use Illuminate\Support\Facades\Cache;

// GET /api/produk — ambil dari cache, kalau tidak ada ambil dari DB
public function index()
{
    $produks = Cache::remember('produk.all', now()->addMinutes(10), function () {
        return Produk::latest()->get();
    });

    return response()->json(['success' => true, 'data' => $produks]);
}

// Hapus cache ketika data berubah
public function store(Request $request)
{
    $produk = Produk::create($request->validated());
    Cache::forget('produk.all');   // invalidasi cache
    return response()->json(['success' => true, 'data' => $produk], 201);
}

public function update(Request $request, Produk $produk)
{
    $produk->update($request->validated());
    Cache::forget('produk.all');   // invalidasi cache
    return response()->json(['success' => true, 'data' => $produk]);
}

public function destroy(Produk $produk)
{
    $produk->delete();
    Cache::forget('produk.all');   // invalidasi cache
    return response()->json(['success' => true]);
}

Metode Cache yang Sering Dipakai

MetodeFungsi
Cache::remember($key, $ttl, $fn)Ambil cache; kalau tidak ada, jalankan $fn dan simpan
Cache::get($key)Ambil nilai dari cache (null jika tidak ada)
Cache::put($key, $value, $ttl)Simpan nilai ke cache
Cache::forget($key)Hapus satu entri cache
Cache::flush()Hapus semua cache
Cache::has($key)Cek apakah key ada di cache
Penting — Cache Invalidation
Selalu hapus cache ketika data di database berubah (store, update, destroy). Cache yang stale lebih berbahaya dari tidak ada cache sama sekali.

Session dengan Redis

Menggunakan Redis sebagai session driver membuat session lebih cepat dan mudah di-scale di multi-server deployment.

.env
SESSION_DRIVER=redis
SESSION_LIFETIME=120   # menit

Tidak ada perubahan kode yang dibutuhkan — Laravel menangani ini secara otomatis. Session tetap bekerja sama seperti sebelumnya dengan session() dan Request::session().

Kapan Pakai Redis Session?
Wajib jika kamu deploy ke lebih dari satu server (load balancing). File session hanya ada di satu server — user yang hit server lain akan logout. Redis session tersentralisasi dan bisa diakses semua server.

Pengenalan Queue (Antrian)

Queue memungkinkan kamu menunda eksekusi task yang memakan waktu ke background, sehingga HTTP response bisa dikembalikan ke user lebih cepat.

Kapan Butuh Queue?

  • Kirim email (bisa lambat jika SMTP server lama merespons)
  • Generate laporan PDF / export Excel besar
  • Resize & upload gambar ke cloud storage
  • Kirim notifikasi ke banyak user sekaligus
  • Integrasi dengan third-party API yang lambat

Alur Kerja Queue

// Tanpa Queue (lambat — user nunggu):
User -> Request -> [kirim email 3 detik] -> Response (lama)

// Dengan Queue (cepat — email dikirim di background):
User -> Request -> [dispatch job ke queue] -> Response (cepat)
                         |
                   [Worker proses job di background]
                         |
                   [Email terkirim]

Membuat Queue Job

Langkah 1 — Konfigurasi Queue Driver

.env
QUEUE_CONNECTION=redis

Langkah 2 — Buat Job Class

Terminal
php artisan make:job SendWelcomeEmail
app/Jobs/SendWelcomeEmail.php
<?php

namespace App\Jobs;

use App\Models\User;
use App\Mail\WelcomeMail;
use Illuminate\Bus\Queueable;
use Illuminate\Contracts\Queue\ShouldQueue;
use Illuminate\Foundation\Bus\Dispatchable;
use Illuminate\Queue\InteractsWithQueue;
use Illuminate\Queue\SerializesModels;
use Illuminate\Support\Facades\Mail;

class SendWelcomeEmail implements ShouldQueue
{
    use Dispatchable, InteractsWithQueue, Queueable, SerializesModels;

    public function __construct(
        public readonly User $user
    ) {}

    public function handle(): void
    {
        Mail::to($this->user->email)
            ->send(new WelcomeMail($this->user));
    }
}

Langkah 3 — Dispatch Job dari Controller

app/Http/Controllers/AuthController.php
use App\Jobs\SendWelcomeEmail;

public function register(Request $request)
{
    // ... validasi dan create user ...
    $user = User::create([...]);

    // Dispatch ke queue — tidak blocking, user langsung dapat response
    SendWelcomeEmail::dispatch($user);

    return response()->json([
        'message' => 'Registrasi berhasil, email konfirmasi dikirim',
        'token'   => $user->createToken('auth_token')->plainTextToken,
    ], 201);
}

Langkah 4 — Jalankan Worker

Terminal
# Jalankan worker (proses jobs dari queue)
php artisan queue:work

# Worker dengan Redis, restart otomatis jika ada file baru
php artisan queue:work redis --tries=3 --timeout=60
Opsi Dispatch Berguna
SendWelcomeEmail::dispatch($user)->delay(now()->addMinutes(5)); — tunda eksekusi job 5 menit.
SendWelcomeEmail::dispatchSync($user); — jalankan langsung (synchronous, tanpa queue — berguna saat testing).

Install Laravel Horizon

Laravel Horizon adalah dashboard monitoring untuk queue berbasis Redis. Kamu bisa melihat jobs yang berjalan, gagal, throughput, dan mengatur worker pool.

Langkah 1 — Install via Composer

Terminal
composer require laravel/horizon

Langkah 2 — Publish Assets

Terminal
php artisan horizon:install

Ini membuat file config/horizon.php dan assets untuk dashboard.

Langkah 3 — Konfigurasi Worker

config/horizon.php
'environments' => [
    'local' => [
        'supervisor-1' => [
            'maxProcesses'  => 3,
            'balanceMaxShift' => 1,
            'balanceCooldown' => 3,
        ],
    ],
    'production' => [
        'supervisor-1' => [
            'maxProcesses'  => 10,
        ],
    ],
],

Langkah 4 — Jalankan Horizon

Terminal
php artisan horizon

Horizon Dashboard

Akses dashboard di http://localhost:8000/horizon setelah Horizon berjalan.

Fitur Dashboard

FiturKeterangan
MonitoringLihat jobs yang sedang berjalan, pending, dan selesai
Failed JobsLihat jobs yang gagal beserta pesan error-nya
RetryCoba ulang job yang gagal dengan satu klik
ThroughputGrafik berapa jobs diproses per menit
RuntimeBerapa lama tiap job rata-rata diproses

Proteksi Dashboard (Production)

app/Providers/AppServiceProvider.php
use Laravel\Horizon\Horizon;

public function boot(): void
{
    Horizon::auth(function ($request) {
        return $request->user() && $request->user()->isAdmin();
    });
}
Perhatian
Di production, selalu batasi akses dashboard hanya untuk admin. Secara default, dashboard hanya bisa diakses di environment local.

Retry Failed Jobs via Artisan

Terminal
# Lihat semua failed jobs
php artisan queue:failed

# Retry satu job berdasarkan ID
php artisan queue:retry {id}

# Retry semua failed jobs
php artisan queue:retry all

# Hapus semua failed jobs
php artisan queue:flush

Pengenalan Testing

Testing memastikan kode kamu bekerja sesuai ekspektasi — sekarang dan setelah ada perubahan di masa depan. Laravel mendukung dua framework testing populer:

FrameworkSintaksKelebihan
Pest Functional, ringkas Lebih sedikit boilerplate, readable, plugin ecosystem bagus
PHPUnit OOP class-based Standard PHP, familiar, Pest berjalan di atasnya

Jenis Test

  • Unit Test — test satu fungsi/method secara terisolasi (tanpa DB, tanpa HTTP)
  • Feature Test — test alur lengkap: HTTP request ke response (boleh hit DB)
Aturan Praktis
Tulis Feature Test untuk endpoint API — pastikan register, login, CRUD produk semuanya menghasilkan response yang benar. Unit Test untuk logic bisnis kompleks yang bisa diisolasi.

Install Pest

Laravel 12 sudah include Pest

Project Laravel 12 baru sudah menyertakan Pest secara default. Jika project kamu masih menggunakan PHPUnit murni, install Pest dengan:

Terminal
composer require pestphp/pest pestphp/pest-plugin-laravel --dev
php artisan pest:install

Struktur Folder Test

tests/
  Feature/       # Feature tests (HTTP, DB)
    Auth/
      AuthTest.php
    Api/
      ProdukApiTest.php
  Unit/           # Unit tests (isolated logic)
    ProdukTest.php
  Pest.php        # Bootstrap Pest

Konfigurasi Database Test

phpunit.xml
<env name="DB_CONNECTION" value="sqlite"/>
<env name="DB_DATABASE" value=":memory:"/>

Ini membuat test menggunakan SQLite in-memory — lebih cepat dan tidak mencemari database development kamu.

Jalankan Test

Terminal
# Jalankan semua test
php artisan test

# Atau langsung via Pest
./vendor/bin/pest

# Jalankan test tertentu
./vendor/bin/pest tests/Feature/Api/ProdukApiTest.php

# Tampilkan detail tiap test
./vendor/bin/pest --verbose

Unit Test dengan Pest

Unit test berjalan cepat karena tidak menyentuh database atau HTTP.

tests/Unit/ProdukTest.php
<?php

it('harga produk harus lebih dari nol', function () {
    $produk = new Produk(['harga' => 0]);

    expect($produk->harga)->toBe(0)
        ->not()->toBeGreaterThan(0);
});

it('nama produk tidak boleh kosong', function () {
    $produk = new Produk();
    expect($produk->nama_produk)->toBeNull();
});

it('produk dapat dihitung stoknya', function () {
    $produk = new Produk(['stok' => 10]);
    expect($produk->stok)->toBe(10);
});

Matcher Pest yang Sering Dipakai

toEqual($val)Loose equal (==), cocok untuk array/object
toBeTrue() / toBeFalse()Cek nilai boolean
toBeNull()Nilai harus null
toBeEmpty()String/array/collection kosong
toContain($val)Array/string mengandung nilai
toHaveCount($n)Array/collection punya N elemen
not()->toBe($val)Negasi dari matcher apapun

Feature Test dengan Pest

Feature test mensimulasikan HTTP request ke aplikasi kamu. Gunakan trait RefreshDatabase agar database di-reset tiap test.

tests/Feature/Auth/AuthTest.php
<?php

use App\Models\User;
use Illuminate\Foundation\Testing\RefreshDatabase;

uses(RefreshDatabase::class);

it('user dapat register dengan data valid', function () {
    $response = $this->postJson('/api/register', [
        'name'                  => 'John Doe',
        'email'                 => 'john@example.com',
        'password'              => 'password123',
        'password_confirmation'  => 'password123',
    ]);

    $response
        ->assertStatus(201)
        ->assertJsonStructure(['message', 'token', 'user']);

    expect(User::count())->toBe(1);
});

it('register gagal jika email sudah dipakai', function () {
    User::factory()->create(['email' => 'john@example.com']);

    $this->postJson('/api/register', [
        'name'                 => 'Jane',
        'email'                => 'john@example.com',
        'password'             => 'password123',
        'password_confirmation' => 'password123',
    ])->assertStatus(422);
});

it('user dapat login dengan kredensial benar', function () {
    User::factory()->create([
        'email'    => 'john@example.com',
        'password' => bcrypt('password123'),
    ]);

    $this->postJson('/api/login', [
        'email'    => 'john@example.com',
        'password' => 'password123',
    ])
        ->assertOk()
        ->assertJsonStructure(['token', 'user']);
});
tests/Feature/Api/ProdukApiTest.php
<?php

use App\Models\{Produk, User};
use Illuminate\Foundation\Testing\RefreshDatabase;

uses(RefreshDatabase::class);

// Helper: buat user + token untuk test
beforeEach(function () {
    $this->user  = User::factory()->create();
    $this->token = $this->user->createToken('test')->plainTextToken;
    $this->headers = ['Authorization' => 'Bearer ' . $this->token];
});

it('dapat mengambil daftar produk', function () {
    Produk::factory()->count(3)->create();

    $this->getJson('/api/produk', $this->headers)
        ->assertOk()
        ->assertJsonCount(3, 'data');
});

it('dapat menambah produk baru', function () {
    $this->postJson('/api/produk', [
        'nama_produk' => 'Laptop Gaming',
        'harga'       => 15000000,
        'stok'        => 5,
    ], $this->headers)
        ->assertStatus(201)
        ->assertJsonPath('data.nama_produk', 'Laptop Gaming');

    expect(Produk::count())->toBe(1);
});

it('tidak bisa akses produk tanpa token', function () {
    $this->getJson('/api/produk')->assertStatus(401);
});

it('dapat menghapus produk', function () {
    $produk = Produk::factory()->create();

    $this->deleteJson("/api/produk/{$produk->id}", [], $this->headers)
        ->assertOk();

    expect(Produk::count())->toBe(0);
});

PHPUnit (Gaya Klasik)

PHPUnit menggunakan class yang extend TestCase. Pest sebenarnya berjalan di atas PHPUnit, jadi kamu bisa mix keduanya.

tests/Feature/Api/ProdukApiTestClass.php
<?php

namespace Tests\Feature\Api;

use App\Models\{Produk, User};
use Illuminate\Foundation\Testing\RefreshDatabase;
use Tests\TestCase;

class ProdukApiTestClass extends TestCase
{
    use RefreshDatabase;

    private User $user;
    private string $token;

    protected function setUp(): void
    {
        parent::setUp();
        $this->user  = User::factory()->create();
        $this->token = $this->user->createToken('test')->plainTextToken;
    }

    public function test_dapat_mengambil_daftar_produk(): void
    {
        Produk::factory()->count(3)->create();

        $this->getJson('/api/produk', [
            'Authorization' => 'Bearer ' . $this->token,
        ])->assertOk()
           ->assertJsonCount(3, 'data');
    }

    public function test_validasi_gagal_jika_nama_kosong(): void
    {
        $this->postJson('/api/produk', ['harga' => 5000], [
            'Authorization' => 'Bearer ' . $this->token,
        ])->assertStatus(422)
           ->assertJsonValidationErrors(['nama_produk']);
    }
}

Pest vs PHPUnit — Perbandingan

AspekPestPHPUnit
Sintaksit('desc', fn() => ...)public function test_desc()
BoilerplateMinimalClass + namespace + setUp
Assertionexpect($val)->toBe(1)$this->assertSame(1, $val)
Groupingdescribe('group', fn)Class method
SetupbeforeEach(fn)setUp()
DirekomendasikanProject baru / modernTim yang familiar PHPUnit
Rekomendasi
Gunakan Pest untuk project baru — sintaksnya lebih bersih dan cepat ditulis. Tapi kalau tim sudah familiar dengan PHPUnit, keduanya sama-sama bagus karena Pest berjalan di atas PHPUnit.

Ringkasan Minggu 3

Kamu sudah mempelajari tiga topik utama optimasi dan otomatisasi Laravel:

  • 1

    Redis

    Cache query lambat dengan Cache::remember(), gunakan Redis sebagai session driver untuk deployment multi-server, invalidasi cache saat data berubah.

  • 2

    Laravel Horizon (Queue)

    Pindahkan task berat (email, export) ke background dengan dispatch(), monitor semua jobs via dashboard Horizon, retry jobs yang gagal.
