Capstone Project: E-commerce

Bukan topik baru — minggu ini menggabungkan semua skill dari Minggu 1–6 untuk membangun satu fitur nyata: alur belanja online lengkap di atas laravel-crud-app, mulai dari keranjang belanja sampai pembayaran terverifikasi.

Apa yang Dibangun

FiturDeskripsi
Shopping CartTambah/ubah/hapus item, cart tersimpan per user
CheckoutValidasi stok, ubah isi cart jadi Order, kurangi stok produk
Payment GatewayIntegrasi Midtrans untuk pembayaran, webhook update status otomatis
Order ManagementStatus order (pending → paid → shipped → completed), endpoint admin

Alur Belanja End-to-End

1. User tambah Produk ke Cart          → CartController::store()
2. User checkout                        → OrderController::store()
   → validasi stok tiap item cart
   → buat Order + OrderItems dari Cart
   → kurangi stok Produk
   → cart dikosongkan
3. Redirect ke Payment Gateway (Midtrans) → PaymentController::createSnapToken()
4. User bayar di halaman Midtrans
5. Midtrans kirim webhook ke server      → PaymentController::webhook()
   → verifikasi signature (Minggu 6)
   → update status Order jadi "paid"
   → broadcast event (Minggu 5) → notifikasi real-time ke admin
Kenapa Ini Disebut Capstone?
Setiap langkah di atas memakai skill dari minggu-minggu sebelumnya: Eloquent & relasi (M1), Repository/Service & otorisasi (M2), cache & queue untuk proses berat (M3), logging transaksi (M4), broadcasting status (M5), dan validasi/signature aman (M6). Tidak ada konsep baru — ini latihan merangkai semuanya.

Kenapa harga_saat_beli Disimpan Terpisah?
Harga produk bisa berubah kapan saja. order_items.harga_saat_beli membekukan harga pada saat transaksi terjadi — supaya riwayat order tetap akurat meski harga produk di master data berubah di kemudian hari.

app/Models/Cart.php
class Cart extends Model
{
    protected $fillable = ['user_id'];

    public function items()
    {
        return $this->hasMany(CartItem::class);
    }

    public function total(): int
    {
        return $this->items->sum(fn($item) => $item->produk->harga * $item->qty);
    }
}

Cart Service

Mengikuti Repository/Service Pattern dari Minggu 2 — logic bisnis cart (validasi stok, gabung qty produk yang sama) ada di Service, bukan di controller.

app/Services/CartService.php
<?php

namespace App\Services;

use App\Models\{Cart, Produk, User};
use Illuminate\Validation\ValidationException;

class CartService
{
    public function getOrCreateCart(User $user): Cart
    {
        return Cart::firstOrCreate(['user_id' => $user->id]);
    }

    public function addItem(User $user, int $produkId, int $qty): Cart
    {
        $produk = Produk::findOrFail($produkId);

        if ($produk->stok < $qty) {
            throw ValidationException::withMessages([
                'qty' => "Stok {$produk->nama_produk} hanya tersisa {$produk->stok}.",
            ]);
        }

        $cart = $this->getOrCreateCart($user);

        $cart->items()->updateOrCreate(
            ['produk_id' => $produkId],
            ['qty' => \DB::raw("qty + {$qty}")]
        );

        return $cart->fresh('items.produk');
    }

    public function removeItem(User $user, int $produkId): Cart
    {
        $cart = $this->getOrCreateCart($user);
        $cart->items()->where('produk_id', $produkId)->delete();

        return $cart->fresh('items.produk');
    }
}

Cart API Endpoints

app/Http/Controllers/Api/CartController.php
class CartController extends Controller
{
    public function __construct(private readonly CartService $cartService) {}

    public function show(Request $request)
    {
        $cart = $this->cartService->getOrCreateCart($request->user())->load('items.produk');
        return response()->json(['success' => true, 'data' => $cart, 'total' => $cart->total()]);
    }

    public function store(Request $request)
    {
        $validated = $request->validate([
            'produk_id' => 'required|exists:produks,id',
            'qty'       => 'required|integer|min:1',
        ]);

        $cart = $this->cartService->addItem($request->user(), $validated['produk_id'], $validated['qty']);
        return response()->json(['success' => true, 'data' => $cart]);
    }

    public function destroy(Request $request, int $produkId)
    {
        $cart = $this->cartService->removeItem($request->user(), $produkId);
        return response()->json(['success' => true, 'data' => $cart]);
    }
}

Dari Cart ke Order (Checkout)

Checkout adalah proses yang harus atomik — kurangi stok, buat order, kosongkan cart, semuanya berhasil bersama atau gagal bersama. Gunakan DB::transaction().

app/Services/OrderService.php
<?php

namespace App\Services;

use App\Models\{Cart, Order, User};
use Illuminate\Support\Facades\DB;
use Illuminate\Validation\ValidationException;

class OrderService
{
    public function checkout(User $user, Cart $cart): Order
    {
        if ($cart->items->isEmpty()) {
            throw ValidationException::withMessages(['cart' => 'Keranjang masih kosong.']);
        }

        return DB::transaction(function () use ($user, $cart) {
            // 1. Validasi stok semua item dulu, sebelum ada yang diubah
            foreach ($cart->items as $item) {
                if ($item->produk->stok < $item->qty) {
                    throw ValidationException::withMessages([
                        'stok' => "Stok {$item->produk->nama_produk} tidak cukup.",
                    ]);
                }
            }

            // 2. Buat order
            $order = Order::create([
                'user_id'   => $user->id,
                'kode_order' => 'ORD-' . strtoupper(uniqid()),
                'status'     => 'pending',
                'total'      => $cart->total(),
            ]);

            // 3. Salin item cart → order_items, kurangi stok
            foreach ($cart->items as $item) {
                $order->items()->create([
                    'produk_id'        => $item->produk_id,
                    'qty'              => $item->qty,
                    'harga_saat_beli'  => $item->produk->harga,
                ]);

                $item->produk->decrement('stok', $item->qty);
            }

            // 4. Kosongkan cart
            $cart->items()->delete();

            return $order;
        });
    }
}
Kenapa Wajib DB::transaction()?
Tanpa transaction, kalau server crash di tengah proses (misal setelah stok dikurangi tapi sebelum order tersimpan), data jadi tidak konsisten — stok berkurang tapi tidak ada order-nya. Transaction menjamin semua langkah sukses bersama atau di-rollback semuanya.

Order Status & State Machine

Status order tidak boleh berubah sembarangan — order yang sudah completed tidak boleh balik jadi pending.

StatusArtiTransisi Berikutnya yang Valid
pendingOrder dibuat, menunggu pembayaranpaid, cancelled
paidPembayaran terverifikasi (via webhook)processing, cancelled
processingSedang disiapkan/dikemasshipped
shippedDalam pengirimancompleted
completedDiterima pembeli, selesai— (final)
cancelledDibatalkan, stok dikembalikan— (final)
app/Models/Order.php
private const TRANSITIONS = [
    'pending'    => ['paid', 'cancelled'],
    'paid'       => ['processing', 'cancelled'],
    'processing' => ['shipped'],
    'shipped'    => ['completed'],
];

public function transitionTo(string $status): void
{
    $allowed = self::TRANSITIONS[$this->status] ?? [];

    if (!in_array($status, $allowed, true)) {
        throw new \DomainException(
            "Tidak bisa ubah status dari '{$this->status}' ke '{$status}'."
        );
    }

    $this->update(['status' => $status]);
}
State Machine Sederhana Tapi Efektif
Tidak perlu package tambahan — array transisi + satu method sudah cukup mencegah bug klasik seperti "order completed tiba-tiba balik pending" akibat request yang salah urutan.

public function updateStatus(Request $request, Order $order)
{
    // hanya admin yang boleh ubah status manual (lihat Minggu 2 — Spatie)
    $this->authorize('update', $order);

    $request->validate(['status' => 'required|in:processing,shipped,completed,cancelled']);
    $order->transitionTo($request->status);

    return response()->json(['success' => true, 'data' => $order]);
}

Buat Snap Token untuk Order

app/Services/PaymentService.php
<?php

namespace App\Services;

use App\Models\{Order, Payment};
use Midtrans\Config;
use Midtrans\Snap;

class PaymentService
{
    public function __construct()
    {
        Config::$serverKey    = config('services.midtrans.server_key');
        Config::$isProduction = config('services.midtrans.is_production');
        Config::$isSanitized  = true;
        Config::$is3ds        = true;
    }

    public function createSnapToken(Order $order): string
    {
        $params = [
            'transaction_details' => [
                'order_id'     => $order->kode_order,
                'gross_amount' => $order->total,
            ],
            'customer_details' => [
                'first_name' => $order->user->name,
                'email'      => $order->user->email,
            ],
        ];

        $snapToken = Snap::getSnapToken($params);

        Payment::create([
            'order_id'           => $order->id,
            'midtrans_order_id'  => $order->kode_order,
            'status'             => 'pending',
        ]);

        return $snapToken;
    }
}
app/Http/Controllers/Api/PaymentController.php
public function createSnapToken(Order $order)
{
    $this->authorize('view', $order);

    $token = $this->paymentService->createSnapToken($order);
    return response()->json(['success' => true, 'snap_token' => $token]);
}

Webhook & Verifikasi Signature

Midtrans memberi tahu status pembayaran lewat webhook (server Midtrans yang mengirim POST ke server kita). Endpoint ini harus publik (tanpa auth:sanctum), tapi tetap wajib diverifikasi keasliannya (prinsip keamanan dari Minggu 6).

app/Http/Controllers/Api/PaymentController.php
public function webhook(Request $request)
{
    $payload = $request->all();

    // Verifikasi signature — pastikan request benar-benar dari Midtrans, bukan penyerang
    $signature = hash('sha512',
        $payload['order_id'] .
        $payload['status_code'] .
        $payload['gross_amount'] .
        config('services.midtrans.server_key')
    );

    if ($signature !== $payload['signature_key']) {
        Log::channel('payment')->warning('Signature webhook tidak valid', $payload);
        abort(403, 'Invalid signature.');
    }

    $payment = Payment::where('midtrans_order_id', $payload['order_id'])->firstOrFail();
    $payment->update([
        'status'       => $payload['transaction_status'],
        'raw_response' => $payload,
    ]);

    if ($payload['transaction_status'] === 'settlement') {
        $payment->order->transitionTo('paid');
        OrderStatusUpdated::dispatch($payment->order);   // broadcast real-time, Minggu 5
        Log::channel('payment')->info('Pembayaran berhasil', ['order_id' => $payment->order_id]);
    }

    return response()->json(['success' => true]);
}
routes/api.php
// Publik (dipanggil server Midtrans, bukan user) — tapi diverifikasi signature
Route::post('/payment/webhook', [PaymentController::class, 'webhook'])
    ->middleware('throttle:30,1');
Endpoint Publik ≠ Endpoint Tanpa Proteksi
Webhook harus bisa diakses tanpa token Sanctum (server Midtrans tidak punya token user), tapi tetap wajib diverifikasi lewat signature. Tanpa ini, siapapun bisa memalsukan callback "pembayaran berhasil" tanpa benar-benar membayar.
