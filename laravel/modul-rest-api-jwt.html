Konsep JWT — Baca Sebelum Mulai

JWT (JSON Web Token) adalah token berbentuk string yang menyimpan informasi user di dalamnya. Berbeda dengan Sanctum yang menyimpan token di database, JWT bersifat stateless — server tidak perlu mengecek database untuk memvalidasi token.

Struktur Token JWT

Token JWT terdiri dari 3 bagian, dipisah oleh titik (.):

eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9
.
eyJzdWIiOjEsImV4cCI6MTcyMDAwMzYwMH0
.
SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV
BagianIsiCatatan
HeaderAlgoritma & tipe tokenBase64 encoded, bisa dibaca siapapun
PayloadData user (id, expiry, dll)Base64 encoded, bisa dibaca siapapun
SignatureHash kriptografiTidak bisa dipalsukan tanpa JWT_SECRET
💡 Kenapa Aman?

Meski payload bisa dibaca, payload tidak bisa diubah. Kalau payload diubah, signature-nya tidak cocok lagi → server tolak token. Ibaratnya seperti segel pada amplop: orang bisa baca tulisan di luar, tapi kalau segel rusak, tahu sudah dibuka.

Alur JWT di Laravel

1. Login  →  Auth::guard('api')->attempt(['email'=>..., 'password'=>...])
              ↓ cek password di database
              ↓ buat JWT token (signed dengan JWT_SECRET)
           ← return token string ke client

2. Request →  GET /api/v2/produk
              Header: Authorization: Bearer eyJhbGci...
              ↓ middleware auth:api
              ↓ verify signature JWT (TANPA query database)
              ↓ extract user ID dari payload
              ↓ load user dari database (1 query)
           ← return data produk
📐 JWT vs Sanctum — Singkat
SanctumJWT
Token disimpan di DB (2 query/request)Token tidak disimpan di DB (1 query/request)
Revoke mudah: hapus row DBAda fitur refresh token bawaan
Cocok: SPA, app sederhanaCocok: mobile app, microservices, multi-platform

Register Service Provider
Daftarkan JWT ke dalam framework Laravel 12

Di Laravel 12, package yang tidak auto-discover harus didaftarkan manual. Buka file bootstrap/providers.php dan tambahkan satu baris:

✎ Edit file: bootstrap/providers.php
<?php

use App\Providers\AppServiceProvider;

return [
    AppServiceProvider::class,
    PHPOpenSourceSaver\JWTAuth\Providers\LaravelServiceProvider::class,  // ← tambahkan baris ini
];
Jangan lupa koma setelah AppServiceProvider::class
PHP array membutuhkan koma di antara setiap elemen.
💡 Apa itu Service Provider?
Service Provider adalah "bootstrapper" — kode yang dijalankan saat Laravel pertama kali start. JWT Service Provider mendaftarkan driver jwt ke dalam sistem autentikasi Laravel sehingga bisa digunakan di config/auth.php.

PENTING — Jangan commit .env ke Git!

💡 Apa itu JWT_SECRET?
Secret ini digunakan untuk membuat Signature (bagian ketiga token JWT). Server menggunakan secret ini untuk memverifikasi bahwa token tidak diubah oleh siapapun. Hanya server yang tahu secret ini.

Buka config/auth.php, temukan bagian 'guards', dan tambahkan guard api:

✎ Edit file: config/auth.php
'guards' => [

        'driver'   => 'session',
        'provider' => 'users',
    ],

    'api' => [             // ← tambahkan blok ini

💡 Apa itu Guard?
Guard adalah cara Laravel mengautentikasi user. Guard web pakai session (untuk website biasa). Guard api yang baru kamu tambahkan pakai driver jwt — digunakan saat kamu menulis middleware auth:api di routes.
5
Update User Model
Implementasikan interface JWTSubject di app/Models/User.php

JWT perlu mengetahui cara mengidentifikasi user. Kamu harus membuat model User mengimplementasikan interface JWTSubject dengan 2 method.

✎ Edit file: app/Models/User.php
<?php

namespace App\Models;

use Database\Factories\UserFactory;
use Illuminate\Database\Eloquent\Factories\HasFactory;
use Illuminate\Foundation\Auth\User as Authenticatable;
use Illuminate\Notifications\Notifiable;
use Laravel\Sanctum\HasApiTokens;
use PHPOpenSourceSaver\JWTAuth\Contracts\JWTSubject;  // ← tambahkan ini

class User extends Authenticatable implements JWTSubject  // ← tambahkan implements
{
    use HasFactory, Notifiable, HasApiTokens;

    protected $fillable = ['name', 'email', 'password'];

    protected $hidden = ['password', 'remember_token'];

    protected function casts(): array
    {
        return [
            'email_verified_at' => 'datetime',
            'password'          => 'hashed',
        ];
    }

    // ─── Tambahkan 2 method ini di bawah ───────────────────

    // Mengembalikan identifier unik user (biasanya primary key)
    public function getJWTIdentifier(): mixed
    {
        return $this->getKey();  // return $this->id
    }

    // Data tambahan yang disimpan di payload JWT
    public function getJWTCustomClaims(): array
    {
        return [];  // kosong = hanya simpan user ID
    }
}
💡 Apa itu Interface JWTSubject?
Interface adalah kontrak — kamu berjanji bahwa class User punya 2 method:
  • getJWTIdentifier() → JWT butuh tahu siapa "subject" token ini (user ID)
  • getJWTCustomClaims() → data tambahan yang mau disimpan di payload (bisa kosong)
Mau simpan role di token? Bisa!
Ubah getJWTCustomClaims() menjadi: 
return ['role' => $this->role];  // role bisa dicek tanpa query DB

6b — Isi controller

Buka file yang baru dibuat dan ganti seluruh isinya:

✎ Edit file: app/Http/Controllers/Api/JwtAuthController.php
<?php

namespace App\Http\Controllers\Api;

use App\Http\Controllers\Controller;
use App\Models\User;
use Illuminate\Http\JsonResponse;
use Illuminate\Http\Request;
use Illuminate\Support\Facades\Auth;
use Illuminate\Support\Facades\Hash;

class JwtAuthController extends Controller
{
    // POST /api/v2/register
    public function register(Request $request): JsonResponse
    {
        $request->validate([
            'name'     => 'required|string|max:255',
            'email'    => 'required|email|unique:users',
            'password' => 'required|min:8|confirmed',
        ]);

        $user = User::create([
            'name'     => $request->name,
            'email'    => $request->email,
            'password' => Hash::make($request->password),
        ]);

        // Langsung login setelah register → dapat JWT token
        $token = Auth::guard('api')->login($user);

        return $this->respondWithToken($token, $user, 201);
    }

    // POST /api/v2/login
    public function login(Request $request): JsonResponse
    {
        // attempt() otomatis cek email + Hash::check password
        // return token jika berhasil, false jika gagal
        $credentials = $request->validate([
            'email'    => 'required|email',
            'password' => 'required',
        ]);

        if (! $token = Auth::guard('api')->attempt($credentials)) {
            return response()->json([
                'success' => false,
                'message' => 'Email atau password salah.',
            ], 401);
        }

        return $this->respondWithToken($token, Auth::guard('api')->user());
    }

    // POST /api/v2/logout
    public function logout(): JsonResponse
    {
        // Masukkan token ke blacklist → tidak bisa dipakai lagi
        Auth::guard('api')->logout();

        return response()->json([
            'success' => true,
            'message' => 'Berhasil logout.',
        ]);
    }

    // POST /api/v2/refresh
    public function refresh(): JsonResponse
    {
        // Invalidasi token lama → buat token baru dengan TTL fresh
        $token = Auth::guard('api')->refresh();

        return $this->respondWithToken($token, Auth::guard('api')->user());
    }

    // GET /api/v2/me
    public function me(): JsonResponse
    {
        return response()->json([
            'success' => true,
            'data'    => Auth::guard('api')->user(),
        ]);
    }

    // Helper — format response token yang konsisten
    protected function respondWithToken(string $token, $user, int $status = 200): JsonResponse
    {
        return response()->json([
            'success'    => true,
            'token'      => $token,
            'token_type' => 'bearer',
            'expires_in' => Auth::guard('api')->factory()->getTTL() * 60,
            'user'       => $user,
        ], $status);
    }
}

SanctumJWT

Buat ProdukApiController
Controller CRUD produk yang return JSON (khusus API)

7a — Generate file controller

▶ Jalankan di terminal
php artisan make:controller Api/ProdukApiController --api

Flag --api membuat controller dengan 5 method (tanpa create dan edit karena API tidak punya form HTML).

7b — Isi controller

✎ Edit file: app/Http/Controllers/Api/ProdukApiController.php
<?php

namespace App\Http\Controllers\Api;

use App\Http\Controllers\Controller;
use App\Models\Produk;
use Illuminate\Http\JsonResponse;
use Illuminate\Http\Request;

class ProdukApiController extends Controller
{
    // GET /api/v2/produk
    public function index(): JsonResponse
    {
        return response()->json([
            'success' => true,
            'data'    => Produk::all(),
        ]);
    }

    // GET /api/v2/produk/{id}
    public function show(Produk $produk): JsonResponse
    {
        return response()->json(['success' => true, 'data' => $produk]);
    }

    // POST /api/v2/produk
    public function store(Request $request): JsonResponse
    {
        $validated = $request->validate([
            'nama_produk' => 'required|string|max:255',
            'harga'       => 'required|numeric|min:1',
            'stok'        => 'required|integer|min:0',
            'deskripsi'   => 'nullable|string|max:500',
        ]);

        $produk = Produk::create($validated);

        return response()->json([
            'success' => true,
            'data'    => $produk,
        ], 201);
    }

    // PUT /api/v2/produk/{id}
    public function update(Request $request, Produk $produk): JsonResponse
    {
        $validated = $request->validate([
            'nama_produk' => 'required|string|max:255',
            'harga'       => 'required|numeric|min:1',
            'stok'        => 'required|integer|min:0',
            'deskripsi'   => 'nullable|string|max:500',
        ]);

        $produk->update($validated);

        return response()->json(['success' => true, 'data' => $produk]);
    }

    // DELETE /api/v2/produk/{id}
    public function destroy(Produk $produk): JsonResponse
    {
        $produk->delete();

        return response()->json([
            'success' => true,
            'message' => 'Produk berhasil dihapus.',
        ]);
    }
}

Update API Routes
Daftarkan semua endpoint JWT di routes/api.php
✎ Edit file: routes/api.php
<?php

use App\Http\Controllers\Api\JwtAuthController;
use App\Http\Controllers\Api\ProdukApiController;
use App\Http\Controllers\AuthController;
use Illuminate\Support\Facades\Route;

// ── Sanctum (v1) ─────────────────────────────────────────
Route::prefix('v1')->group(function () {
    Route::post('/register', [AuthController::class, 'register']);
    Route::post('/login',    [AuthController::class, 'login']);

    Route::middleware('auth:sanctum')->group(function () {
        Route::post('/logout', [AuthController::class, 'logout']);
        Route::get('/me',      [AuthController::class, 'me']);
    });
});

// ── JWT (v2) ──────────────────────────────────────────────
Route::prefix('v2')->group(function () {

    // Public — tidak perlu token
    Route::post('/register', [JwtAuthController::class, 'register']);
    Route::post('/login',    [JwtAuthController::class, 'login']);

    // Protected — harus kirim JWT token di header
    Route::middleware('auth:api')->group(function () {
        Route::post('/logout',  [JwtAuthController::class, 'logout']);
        Route::post('/refresh', [JwtAuthController::class, 'refresh']);
        Route::get('/me',       [JwtAuthController::class, 'me']);

        Route::apiResource('produk', ProdukApiController::class);
    });
});

10
Test Refresh Token
Fitur unik JWT — perbarui token tanpa harus login ulang

Refresh token adalah fitur yang membedakan JWT dari Sanctum. Kamu bisa memperbarui token yang hampir expired tanpa meminta user login ulang.

💡 Konsep TTL (Time To Live)
  • TTL = berapa lama access token berlaku (default: 60 menit)
  • Refresh TTL = berapa lama kamu masih bisa refresh (default: 2 minggu)
  • Selama masih dalam refresh TTL, kamu bisa terus refresh token meski TTL habis

Test Refresh di Postman

  1. Login dulu → dapat token A
  2. Buat request: POST {{base_url}}/api/v2/refresh
  3. Kirim token A di header: Authorization: Bearer {{jwt_token}}
  4. Klik Send → dapat token B baru
  5. Coba pakai token A lagi → 401 Token is invalid (token lama di-blacklist)
  6. Pakai token B → berhasil ✓

Ubah TTL untuk Eksperimen

Buka .env, tambahkan:

JWT_TTL=1        # token expired setelah 1 menit (untuk coba-coba)
JWT_REFRESH_TTL=20160  # refresh window: 2 minggu
▶ Clear config cache setelah ubah .env
php artisan config:clear

Sekarang coba: login → tunggu 1 menit → akses produk → dapat Token has expired → kirim ke /refresh → dapat token baru → akses produk lagi → berhasil!
