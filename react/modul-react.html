React — dari Instalasi sampai Profesional

Modul frontend untuk kategori Programming. Disusun untuk kamu yang sudah pernah belajar JavaScript dan ingin naik ke React — mulai dari memasang tools, memahami component & state, mengambil data dari API Laravel, sampai pola-pola yang dipakai di dunia kerja. Semua contoh kode memakai JavaScript (.jsx).

Apa Itu React & Cara Kerjanya

React adalah library JavaScript untuk membangun antarmuka (UI) berbasis component — potongan UI yang bisa dipakai ulang, seperti "kartu produk", "tombol", atau "navbar". Kamu menyusun aplikasi dari component-component kecil, bukan menulis satu file HTML raksasa.

Kenapa React Dipakai

  • Component reusable — tulis sekali, pakai di banyak tempat.
  • Deklaratif — kamu cukup mendeskripsikan "UI seperti apa untuk data ini", React yang mengurus cara memperbarui tampilan.
  • Ekosistem terbesar — paling banyak library, tutorial, dan lowongan kerja.

Deklaratif vs Imperatif

Dengan JavaScript biasa (imperatif), kamu memberi perintah langkah demi langkah: "cari elemen, ubah teksnya, tambah kelas". Dengan React (deklaratif), kamu bilang "tampilan harus begini kalau datanya begitu" — saat data berubah, React otomatis memperbarui bagian yang perlu saja lewat mekanisme Virtual DOM (salinan ringan struktur halaman yang dipakai React untuk menghitung perubahan seminimal mungkin).

React itu "Library", Bukan "Framework"
Berbeda dengan Laravel yang serba lengkap, React hanya mengurus tampilan. Untuk routing, pemanggilan API, dan state global, kamu memasang library tambahan sesuai kebutuhan — itulah yang akan kita pelajari bertahap di modul ini.

Persiapan Tools di Windows

Kabar baik untuk pengguna Windows: React tidak butuh aplikasi berat seperti XAMPP. Yang benar-benar wajib dipasang cuma satu: Node.js.

1. Pasang Node.js

Unduh installer LTS dari nodejs.org, jalankan, lalu klik Next sampai selesai (biarkan opsi default). npm ikut terpasang otomatis. Verifikasi lewat terminal (PowerShell atau Git Bash):

Terminal
node -v
npm -v

Kalau keduanya menampilkan nomor versi (mis. v20.x dan 10.x), Node.js siap.

2. Extension VS Code yang Membantu (opsional)

  • ES7+ React/Redux snippets — pintasan menulis component cepat.
  • Prettier — merapikan format kode otomatis saat menyimpan.
Cara Membuktikan Tahap Ini Berhasil
Buka terminal dan jalankan node -v. Jika muncul versi (bukan pesan "'node' is not recognized"), berarti Node.js & npm sudah terpasang dengan benar dan kamu siap membuat project React pertama.

Project Pertama dengan Vite

Vite adalah tool modern untuk membuat & menjalankan project React dengan cepat (menggantikan Create React App yang kini sudah tidak dianjurkan). Buat project baru lewat terminal:

Terminal
npm create vite@latest toko-react -- --template react
cd toko-react
npm install
npm run dev

Perintah terakhir menjalankan development server. Buka alamat yang muncul (biasanya http://localhost:5173) di browser — kamu akan lihat halaman awal React. Setiap kali kamu menyimpan perubahan file, halaman ikut ter-update otomatis (hot reload).

Struktur Folder Penting

toko-react/
  node_modules/      library (jangan diutak-atik, jangan di-commit)
  public/            aset statis (gambar, favicon)
  src/               <-- di sinilah kamu bekerja
    main.jsx         titik masuk aplikasi
    App.jsx          component utama
    App.css
  index.html         satu-satunya file HTML
  package.json       daftar dependency & script

Isi src/App.jsx kira-kira seperti ini — ganti seluruh isinya untuk mencoba:

src/App.jsx
function App() {
  return (
    <div>
      <h1>Toko React</h1>
      <p>Halaman pertamaku dengan React.</p>
    </div>
  );
}

export default App;
Kenapa Ada node_modules yang Besar Sekali?
Folder itu berisi semua library yang dibutuhkan project. Ukurannya bisa ratusan MB, tapi tidak perlu di-commit ke Git — cukup file package.json. Orang lain tinggal menjalankan npm install untuk membuat ulang folder itu. Pastikan node_modules ada di .gitignore.
Cara Membuktikan Tahap Ini Berhasil
Setelah npm run dev, buka http://localhost:5173. Ubah tulisan di App.jsx lalu simpan — teks di browser harus berubah otomatis tanpa kamu me-refresh. Kalau itu terjadi, lingkungan React-mu sudah berjalan sempurna.

JSX — Menulis HTML di Dalam JavaScript

JSX adalah sintaks yang membuatmu bisa menulis tampilan mirip HTML langsung di dalam JavaScript. Inilah ciri khas React yang di awal terasa asing, tapi cepat jadi terbiasa.

JSX
function Salam() {
  const nama = "Rahmat";
  return <h1>Halo, {nama}!</h1>;
}

Kurung kurawal { } adalah "jendela" untuk menyisipkan nilai JavaScript ke dalam tampilan — bisa variabel, hasil perhitungan, atau pemanggilan fungsi.

Beda JSX dari HTML Biasa

HTMLJSXAlasan
class="..."className="..."class kata khusus di JavaScript
onclick="..."onClick={...}penulisan camelCase & pakai fungsi
<br><br />semua tag harus ditutup
Aturan: Satu Elemen Pembungkus
Sebuah component harus mengembalikan satu elemen induk. Kalau butuh beberapa elemen sejajar tanpa menambah <div>, bungkus dengan Fragment kosong: <> ... </>.

Component & Props

Component adalah fungsi JavaScript yang mengembalikan JSX. Namanya wajib diawali huruf besar (KartuProduk, bukan kartuProduk).

Props adalah cara mengoper data dari component induk ke component anak — mirip "atribut" pada tag HTML. Ini membuat satu component bisa dipakai ulang dengan isi berbeda.

src/components/KartuProduk.jsx
function KartuProduk({ nama, harga }) {
  return (
    <div className="kartu">
      <h3>{nama}</h3>
      <p>Rp{harga}</p>
    </div>
  );
}

export default KartuProduk;

Memakainya di component lain — setiap pemanggilan mengoper props berbeda:

src/App.jsx
import KartuProduk from "./components/KartuProduk";

function App() {
  return (
    <div>
      <KartuProduk nama="Sepatu Lari" harga={250000} />
      <KartuProduk nama="Tas Ransel" harga={180000} />
    </div>
  );
}
Props Bersifat Read-Only
Component anak tidak boleh mengubah props yang diterimanya. Props itu "milik" induk — anak hanya membacanya. Kalau anak butuh data yang bisa berubah, itulah gunanya state (bagian berikutnya).

Rendering List & Conditional

Menampilkan Daftar dengan .map()

Untuk menampilkan banyak item dari sebuah array, gunakan .map() — method JavaScript yang sudah kamu kenal. Tiap item wajib punya prop key yang unik supaya React efisien melacak perubahan.

JSX
const produk = [
  { id: 1, nama: "Sepatu" },
  { id: 2, nama: "Tas" },
  { id: 3, nama: "Topi" },
];

function DaftarProduk() {
  return (
    <ul>
      {produk.map((p) => (
        <li key={p.id}>{p.nama}</li>
      ))}
    </ul>
  );
}

Conditional Rendering

Menampilkan sesuatu hanya jika kondisi terpenuhi:

JSX
{isLogin ? <p>Selamat datang</p> : <p>Silakan login</p>}

{stok > 0 && <span>Stok tersedia</span>}

Operator ? : untuk memilih antara dua tampilan; operator && untuk menampilkan sesuatu hanya kalau kondisinya benar.

Jangan Pakai Index Array sebagai key
Menulis key={index} tampak gampang, tapi bikin bug saat daftar diurutkan atau item dihapus. Gunakan id unik dari data (mis. p.id).

State (useState) & Event

State adalah data yang bisa berubah dari dalam component dan membuat tampilan ikut diperbarui. Kita mengaksesnya lewat hook bernama useState.

src/components/Counter.jsx
import { useState } from "react";

function Counter() {
  const [jumlah, setJumlah] = useState(0);

  return (
    <button onClick={() => setJumlah(jumlah + 1)}>
      Klik saya: {jumlah}
    </button>
  );
}

useState(0) membuat state dengan nilai awal 0, dan mengembalikan dua hal: nilai sekarang (jumlah) dan fungsi untuk mengubahnya (setJumlah). Setiap kali setJumlah dipanggil, React me-render ulang component dengan nilai baru.

Jangan Ubah State Secara Langsung
Menulis jumlah = jumlah + 1 tidak akan memperbarui tampilan. Perubahan state harus lewat fungsi setter (setJumlah) supaya React tahu ada yang berubah dan perlu me-render ulang.
Cara Membuktikan Tahap Ini Berhasil
Tampilkan <Counter /> di App.jsx, jalankan, lalu klik tombolnya. Angka harus bertambah setiap klik tanpa halaman ter-refresh. Itu bukti state & re-render bekerja.

Form & Lifting State Up

Controlled Component

Di React, nilai input form biasanya "dikendalikan" oleh state — disebut controlled component. Nilai input datang dari state, dan setiap ketikan memperbarui state itu.

JSX
import { useState } from "react";

function FormNama() {
  const [nama, setNama] = useState("");

  return (
    <div>
      <input
        value={nama}
        onChange={(e) => setNama(e.target.value)}
        placeholder="Ketik nama"
      />
      <p>Halo, {nama}</p>
    </div>
  );
}

Lifting State Up

Kalau dua component bersaudara perlu berbagi data yang sama, angkat state-nya ke induk terdekat, lalu turunkan lewat props. Prinsipnya: state hidup di tempat tertinggi yang membutuhkannya, lalu mengalir ke bawah.

Data Mengalir Satu Arah (One-Way Data Flow)
Di React, data selalu mengalir dari induk ke anak lewat props. Kalau anak perlu memberitahu induk, induk mengoper sebuah fungsi lewat props yang dipanggil anak — bukan anak yang mengubah data induk langsung.

useEffect — Efek Samping

useEffect dipakai untuk menjalankan kode "efek samping" — hal yang terjadi di luar sekadar menampilkan UI: mengambil data dari server, memasang timer, atau menyimpan ke localStorage.

JSX
import { useEffect, useState } from "react";

function Jam() {
  const [waktu, setWaktu] = useState(new Date());

  useEffect(() => {
    const timer = setInterval(() => setWaktu(new Date()), 1000);
    // fungsi cleanup: dijalankan saat component dilepas
    return () => clearInterval(timer);
  }, []); // [] = jalankan sekali saat pertama muncul

  return <p>{waktu.toLocaleTimeString()}</p>;
}
Arti Array Dependency []
Argumen kedua useEffect menentukan kapan efek dijalankan ulang:
  • [] kosong → sekali saja saat component pertama muncul.
  • [nilai] → setiap kali nilai berubah.
  • tanpa argumen → setiap render (jarang dibutuhkan, sering bikin bug).

Ambil Data dari API

Inilah jembatan React ke backend Laravel-mu: React memanggil REST API, lalu menampilkan hasilnya. Pola paling dasar memakai fetch di dalam useEffect, dengan state terpisah untuk data, loading, dan error.

src/components/DaftarProduk.jsx
import { useEffect, useState } from "react";

function DaftarProduk() {
  const [produk, setProduk] = useState([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    fetch("http://localhost:8000/api/produk")
      .then((res) => {
        if (!res.ok) throw new Error("Gagal memuat");
        return res.json();
      })
      .then((data) => setProduk(data))
      .catch((err) => setError(err.message))
      .finally(() => setLoading(false));
  }, []);

  if (loading) return <p>Memuat...</p>;
  if (error) return <p>Error: {error}</p>;

  return (
    <ul>
      {produk.map((p) => (
        <li key={p.id}>{p.nama} — Rp{p.harga}</li>
      ))}
    </ul>
  );
}

Alternatif: axios

axios adalah library populer yang membuat pemanggilan API lebih ringkas (otomatis mengubah JSON, penanganan error lebih rapi). Pasang dulu: npm install axios.

JSX
import axios from "axios";

const res = await axios.get("http://localhost:8000/api/produk");
console.log(res.data); // langsung berupa data, tanpa .json()
Siapkan CORS di Laravel
Kalau React (port 5173) memanggil Laravel (port 8000), browser akan memblokir karena beda origin. Aktifkan CORS di sisi Laravel (file config/cors.php) agar mengizinkan asal http://localhost:5173.
Cara Membuktikan Tahap Ini Berhasil
Jalankan API Laravel (php artisan serve) dan React bersamaan. Halaman React harus menampilkan "Memuat..." sesaat, lalu berganti jadi daftar produk asli dari database. Cek juga tab Network di DevTools browser: harus ada request ke /api/produk berstatus 200.

TanStack Query (Sekilas)

Pola useEffect + tiga state di atas berulang terus dan cepat berantakan. TanStack Query (dulu React Query) mengurus loading, error, caching, dan refetch secara otomatis — ini cara profesional mengambil data. Pasang: npm install @tanstack/react-query.

JSX
import { useQuery } from "@tanstack/react-query";

function DaftarProduk() {
  const { data, isLoading, error } = useQuery({
    queryKey: ["produk"],
    queryFn: () =>
      fetch("http://localhost:8000/api/produk").then((r) => r.json()),
  });

  if (isLoading) return <p>Memuat...</p>;
  if (error) return <p>Error memuat data</p>;

  return (
    <ul>
      {data.map((p) => (
        <li key={p.id}>{p.nama}</li>
      ))}
    </ul>
  );
}

Bandingkan: tidak ada lagi tiga useState dan useEffect manual. Kamu cukup mendeklarasikan "data apa" dan "cara mengambilnya".

Belajar Bertahap
Untuk pemula, pahami dulu cara manual (fetch + useEffect) supaya mengerti apa yang sebenarnya terjadi. Setelah itu barulah beralih ke TanStack Query untuk proyek nyata — termasuk project e-commerce nanti.

React Router — Navigasi Antar Halaman

React membuat Single Page Application (SPA): satu halaman HTML, tapi terasa seperti banyak halaman. React Router mengatur halaman mana yang tampil berdasarkan URL, tanpa reload. Pasang: npm install react-router-dom.

src/App.jsx
import { BrowserRouter, Routes, Route, Link } from "react-router-dom";

function App() {
  return (
    <BrowserRouter>
      <nav>
        <Link to="/">Beranda</Link>
        <Link to="/produk">Produk</Link>
      </nav>

      <Routes>
        <Route path="/" element={<Beranda />} />
        <Route path="/produk" element={<DaftarProduk />} />
        <Route path="/produk/:id" element={<DetailProduk />} />
      </Routes>
    </BrowserRouter>
  );
}

Gunakan <Link> (bukan <a>) supaya perpindahan halaman tidak me-reload seluruh aplikasi.

Membaca Parameter URL

Untuk /produk/:id, ambil id lewat hook useParams:

src/pages/DetailProduk.jsx
import { useParams } from "react-router-dom";

function DetailProduk() {
  const { id } = useParams();
  // gunakan id untuk fetch /api/produk/{id}
  return <h2>Detail produk #{id}</h2>;
}

Protected Route (Halaman Butuh Login)

Halaman tertentu hanya boleh diakses kalau sudah login. Polanya:

src/components/ProtectedRoute.jsx
import { Navigate } from "react-router-dom";

function ProtectedRoute({ children }) {
  const token = localStorage.getItem("token");
  return token ? children : <Navigate to="/login" />;
}

// dipakai membungkus route:
// <Route path="/dashboard" element={
//   <ProtectedRoute><Dashboard /></ProtectedRoute>
// } />
Keamanan Sebenarnya Ada di Backend
Protected route di frontend hanya menyembunyikan tampilan — bukan pengaman sungguhan. Pengamanan data tetap wajib di Laravel (mis. middleware auth), karena siapa pun bisa mengakali JavaScript di browser.

Context API — State Global Sederhana

Mengoper props berlapis-lapis (induk → anak → cucu) melelahkan — disebut prop drilling. Context API menyediakan data yang bisa diakses component mana pun tanpa dioper manual. Cocok untuk data global seperti info user login atau isi keranjang.

src/context/AuthContext.jsx
import { createContext, useContext, useState } from "react";

const AuthContext = createContext();

export function AuthProvider({ children }) {
  const [user, setUser] = useState(null);
  return (
    <AuthContext.Provider value={{ user, setUser }}>
      {children}
    </AuthContext.Provider>
  );
}

// hook agar pemakaian ringkas
export function useAuth() {
  return useContext(AuthContext);
}

Bungkus aplikasi dengan provider, lalu akses di component mana pun:

JSX
// main.jsx: <AuthProvider><App /></AuthProvider>

// di component mana pun:
const { user, setUser } = useAuth();

Custom Hooks & Zustand

Custom Hook

Custom hook adalah fungsi biasa yang diawali kata use dan membungkus logika reusable (mis. mengambil data). Tujuannya: component jadi bersih, logika bisa dipakai ulang.

src/hooks/useProduk.js
import { useState, useEffect } from "react";

export function useProduk() {
  const [produk, setProduk] = useState([]);

  useEffect(() => {
    fetch("http://localhost:8000/api/produk")
      .then((r) => r.json())
      .then(setProduk);
  }, []);

  return produk;
}

// di component: const produk = useProduk();

Zustand (Sekilas)

Untuk state global yang lebih kompleks, Zustand lebih ringkas daripada Context/Redux. Contoh keranjang belanja:

src/store/cart.js
import { create } from "zustand";

export const useCart = create((set) => ({
  items: [],
  tambah: (produk) =>
    set((state) => ({ items: [...state.items, produk] })),
}));

// di component: const { items, tambah } = useCart();
Kapan Butuh State Management Global?
Jangan buru-buru. Untuk aplikasi kecil, useState + Context sudah cukup. Zustand atau Redux baru berguna saat banyak component jauh berbagi state yang sering berubah (mis. keranjang, notifikasi, tema). Pakai yang paling sederhana yang menyelesaikan masalahmu.

Styling: CSS Module & Tailwind

1. CSS Biasa & CSS Module

Cara paling dasar: buat file .css lalu import. Masalahnya, nama kelas bisa bentrok antar file. CSS Module menyelesaikan ini — nama kelas otomatis dibuat unik per component. Namai file Nama.module.css.

KartuProduk.module.css
.kartu {
  border: 1px solid #ddd;
  border-radius: 8px;
  padding: 16px;
}
KartuProduk.jsx
import styles from "./KartuProduk.module.css";

function KartuProduk() {
  return <div className={styles.kartu}>...</div>;
}

2. Tailwind CSS

Tailwind membuatmu menata gaya lewat kelas siap pakai langsung di JSX — sangat populer di ekosistem React. Pasang di project Vite:

Terminal
npm install -D tailwindcss postcss autoprefixer
npx tailwindcss init -p

Setelah dikonfigurasi, kamu menata gaya dengan kelas utilitas:

JSX
<button className="bg-blue-500 text-white px-4 py-2 rounded hover:bg-blue-600">
  Beli Sekarang
</button>
Pilih Satu, Konsisten
Tidak perlu memakai semua metode. Untuk project e-commerce nanti, Tailwind adalah pilihan yang praktis & cepat. Yang penting: konsisten satu pendekatan di seluruh project agar rapi.

Struktur Folder & Best Practice

Saat project membesar, struktur folder yang rapi bikin kamu (dan tim) tidak tersesat. Susunan umum yang dipakai profesional:

src/
src/
  components/    komponen UI reusable (Button, KartuProduk, Navbar)
  pages/         halaman per route (Beranda, DetailProduk, Dashboard)
  hooks/         custom hooks (useAuth, useProduk)
  services/      fungsi pemanggilan API (api.js, produkService.js)
  context/       state global (AuthContext)
  store/         state management (zustand) bila dipakai
  App.jsx        susunan route
  main.jsx       titik masuk aplikasi

Prinsip Component yang Baik

  • Satu tanggung jawab — satu component fokus satu hal.
  • Kecil & bisa dipakai ulang — pecah UI besar jadi bagian kecil.
  • Pisahkan logika dari tampilan — taruh logika API di services/ atau custom hook, bukan menumpuk di dalam JSX.
  • Penamaan jelas — TombolCheckout lebih baik daripada Btn2.

Pusatkan Konfigurasi API

src/services/api.js
import axios from "axios";

const api = axios.create({
  baseURL: import.meta.env.VITE_API_URL,
});

// sisipkan token otomatis di setiap request
api.interceptors.request.use((config) => {
  const token = localStorage.getItem("token");
  if (token) config.headers.Authorization = `Bearer ${token}`;
  return config;
});

export default api;

Performa

React sudah cepat secara default. Optimasi di bawah ini baru dipakai kalau memang ada masalah performa yang terukur — bukan dipasang di mana-mana sejak awal.

AlatGunanya
memo()Mencegah component me-render ulang jika props-nya tidak berubah
useMemo()Menyimpan hasil perhitungan berat agar tidak dihitung ulang tiap render
useCallback()Menyimpan referensi fungsi agar stabil antar render
JSX
import { memo, useMemo } from "react";

const KartuProduk = memo(function KartuProduk({ produk }) {
  return <div>{produk.nama}</div>;
});

// perhitungan berat hanya diulang saat `items` berubah:
const total = useMemo(() => hitungTotal(items), [items]);

Error Boundary

Agar satu error tidak membuat seluruh aplikasi blank, bungkus bagian penting dengan Error Boundary. Cara termudah pakai library react-error-boundary.

Jangan Optimasi Terlalu Dini
Menaburkan memo/useMemo di semua tempat justru menambah kerumitan tanpa manfaat nyata. Ukur dulu dengan React DevTools Profiler, baru optimasi bagian yang benar-benar lambat.

Environment Variable & Pengenalan TypeScript

Environment Variable

Jangan menulis alamat API secara hard-code di banyak tempat. Simpan di file .env — di Vite, nama variabel wajib diawali VITE_.

.env
VITE_API_URL=http://localhost:8000/api
Memakainya
const url = import.meta.env.VITE_API_URL;

Saat deploy ke server produksi, cukup ganti nilainya tanpa mengubah kode.

Pengenalan TypeScript

TypeScript adalah JavaScript dengan tipe data. Ia menangkap kesalahan (mis. salah nama properti) sebelum kode dijalankan. Industri makin banyak memakainya. Contoh mendefinisikan bentuk data produk:

TSX
type Produk = {
  id: number;
  nama: string;
  harga: number;
};

function KartuProduk({ produk }: { produk: Produk }) {
  return <h3>{produk.nama}</h3>;
}
Langkah Selanjutnya, Bukan Sekarang
Kuasai dulu React dengan JavaScript. Setelah nyaman, coba project baru dengan template react-ts di Vite. Karena kamu sudah paham React, menambahkan TypeScript nanti terasa jauh lebih ringan.

Build & Deploy

Build untuk Produksi

Selama ini kamu memakai dev server. Untuk mengunggah ke internet, buat versi produksi yang sudah dioptimasi:

Terminal
npm run build     # hasil ada di folder dist/
npm run preview   # coba dulu hasil build secara lokal

Folder dist/ berisi file statis (HTML, CSS, JS) yang siap di-hosting di mana saja.

Deploy

LayananCara singkat
VercelHubungkan repo GitHub → deploy otomatis tiap push
NetlifySeret folder dist/, atau hubungkan repo
GitHub PagesHosting gratis file statis (perlu sedikit konfigurasi base path)
Ganti Alamat API saat Produksi
Di server, localhost:8000 tidak ada. Set VITE_API_URL ke alamat API Laravel yang sudah online, dan pastikan CORS di backend mengizinkan domain frontend produksimu.
Cara Membuktikan Tahap Ini Berhasil
Jalankan npm run build lalu npm run preview. Buka alamat preview — aplikasi harus berjalan sama seperti mode dev, tapi kini dari file di folder dist/. Itu bukti aplikasimu siap di-deploy.
