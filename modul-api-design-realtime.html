Ringkasan Minggu 5

API Design & Real-time

API laravel-crud-app sekarang sudah dipakai banyak client (web, mobile). Minggu ini fokus membuat API tetap stabil untuk client lama saat berkembang (versioning), mudah dipahami tim lain (dokumentasi OpenAPI/Swagger), dan bisa mengirim update secara langsung tanpa polling (real-time dengan Laravel Reverb).

Kenapa Butuh API Versioning?

Begitu API dipakai client eksternal (aplikasi mobile yang sudah di-publish, partner integrasi), kamu tidak bisa lagi bebas mengubah struktur response tanpa merusak mereka.

SkenarioTanpa VersioningDengan Versioning
Ubah nama field nama_produk → nameSemua client lama langsung errorClient lama tetap pakai /v1, yang baru pindah ke /v2
Hapus field yang sudah tidak dipakaiMobile app versi lama crashField tetap ada di /v1, dihapus mulai /v2
Rilis fitur besar yang mengubah alurRollout serentak, berisiko tinggiRollout bertahap, client migrasi di waktunya sendiri
Prinsip Dasar
Sekali sebuah versi API dirilis dan dipakai client, jangan lagi ubah struktur response-nya. Kalau perlu perubahan besar, buat versi baru.

Strategi Versioning

StrategiContohKelebihanKekurangan
URI Path /api/v1/produk Jelas terlihat di URL, mudah di-cache, gampang di-test manual URL berubah setiap versi baru
Header Accept: application/vnd.app.v1+json URL tetap sama, lebih "RESTful murni" Kurang terlihat, lebih sulit di-test lewat browser
Query Parameter /api/produk?version=1 Mudah diimplementasi Gampang lupa disertakan, kurang eksplisit
Rekomendasi
Untuk kebanyakan aplikasi, URI Path versioning (/api/v1/...) paling praktis — mudah dipahami tim, mudah didokumentasikan, dan didukung baik oleh routing Laravel.

Implementasi Versioning di Routes

Struktur Folder Controller per Versi

app/Http/Controllers/Api/
  V1/
    ProdukApiController.php
  V2/
    ProdukApiController.php   # response berbeda dari V1

Routes dengan Prefix Versi

routes/api.php
use App\Http\Controllers\Api\V1\ProdukApiController as ProdukV1;
use App\Http\Controllers\Api\V2\ProdukApiController as ProdukV2;

Route::prefix('v1')->middleware('auth:sanctum')->group(function () {
    Route::apiResource('produk', ProdukV1::class);
});

Route::prefix('v2')->middleware('auth:sanctum')->group(function () {
    Route::apiResource('produk', ProdukV2::class);
});

Contoh Perbedaan Response Antar Versi

V1 — format lama
{
  "success": true,
  "data": { "id": 1, "nama_produk": "Laptop", "harga": 8000000 }
}
V2 — nama field distandarkan ke Inggris + nested pricing
{
  "success": true,
  "data": {
    "id": 1,
    "name": "Laptop",
    "pricing": { "amount": 8000000, "currency": "IDR" }
  }
}

Deprecation Header

Beri tahu client bahwa versi lama akan dihentikan, tanpa langsung mematikannya.

// Middleware khusus V1
public function handle($request, Closure $next)
{
    $response = $next($request);
    $response->header('Deprecation', 'true');
    $response->header('Sunset', '2026-12-31');
    return $response;
}
Jangan Gandakan Semua Logic
Kalau perbedaan V1/V2 cuma di format response, cukup gandakan bagian transformasi data (misal pakai API Resource), bukan seluruh business logic. Service & Repository dari Minggu 2 tetap dipakai bersama.

Install L5-Swagger (OpenAPI)

OpenAPI (dulu disebut Swagger) adalah standar untuk mendeskripsikan REST API secara terstruktur — endpoint, parameter, response — sehingga bisa dibaca manusia (dokumentasi interaktif) maupun mesin (generate SDK client otomatis).

Langkah 1 — Install Package

Terminal
composer require darkaonline/l5-swagger

Langkah 2 — Publish Config

Terminal
php artisan vendor:publish --provider="L5Swagger\L5SwaggerServiceProvider"

Langkah 3 — Info Dasar API

app/Http/Controllers/Controller.php
/**
 * @OA\Info(
 *     title="Laravel CRUD App API",
 *     version="2.0.0",
 *     description="API produk untuk aplikasi laravel-crud-app"
 * )
 * @OA\SecurityScheme(
 *     securityScheme="sanctum",
 *     type="http",
 *     scheme="bearer"
 * )
 */
class Controller extends BaseController
{
    use AuthorizesRequests, ValidatesRequests;
}

Anotasi Endpoint

Tambahkan docblock @OA\... di atas setiap method controller yang ingin didokumentasikan.

app/Http/Controllers/Api/V2/ProdukApiController.php
/**
 * @OA\Get(
 *     path="/api/v2/produk",
 *     summary="Daftar semua produk",
 *     tags={"Produk"},
 *     security={{"sanctum":{}}},
 *     @OA\Response(
 *         response=200,
 *         description="Berhasil",
 *         @OA\JsonContent(
 *             @OA\Property(property="success", type="boolean", example=true),
 *             @OA\Property(property="data", type="array",
 *                 @OA\Items(
 *                     @OA\Property(property="id", type="integer", example=1),
 *                     @OA\Property(property="name", type="string", example="Laptop Gaming"),
 *                 )
 *             )
 *         )
 *     ),
 *     @OA\Response(response=401, description="Token tidak valid")
 * )
 */
public function index()
{
    return response()->json(['success' => true, 'data' => $this->produkService->getAll()]);
}

/**
 * @OA\Post(
 *     path="/api/v2/produk",
 *     summary="Tambah produk baru",
 *     tags={"Produk"},
 *     security={{"sanctum":{}}},
 *     @OA\RequestBody(
 *         required=true,
 *         @OA\JsonContent(
 *             required={"nama_produk","harga","stok"},
 *             @OA\Property(property="nama_produk", type="string", example="Keyboard Mechanical"),
 *             @OA\Property(property="harga", type="integer", example=750000),
 *             @OA\Property(property="stok", type="integer", example=20)
 *         )
 *     ),
 *     @OA\Response(response=201, description="Produk berhasil dibuat"),
 *     @OA\Response(response=422, description="Validasi gagal")
 * )

public function store(StoreProdukRequest $request) { /* ... */ }

Generate & Lihat Dokumentasi

Terminal
php artisan l5-swagger:generate

Buka http://127.0.0.1:8000/api/documentation — muncul Swagger UI interaktif: daftar semua endpoint, bisa langsung dicoba ("Try it out") dari browser, lengkap dengan contoh request/response.

ManfaatUntuk Siapa
Coba endpoint langsung dari browserTim frontend/mobile yang mengonsumsi API
Kontrak API yang jelas & terversiTim eksternal/partner integrasi
Generate SDK client otomatisTim yang butuh client library di banyak bahasa
Dokumentasi selalu sinkron dengan kodeSemua tim — docblock ada di sebelah kode aslinya
Auto-regenerate saat Development
L5_SWAGGER_GENERATE_ALWAYS=true di .env membuat dokumentasi ter-generate ulang otomatis setiap kali halaman dokumentasi diakses — matikan opsi ini di production supaya tidak membebani server.

Pengenalan Broadcasting & Laravel Reverb

Sejauh ini, client harus polling (request berulang) untuk tahu kalau ada data baru, misal stok produk berubah. Broadcasting membalik alurnya: server yang aktif mengirim update ke client begitu ada perubahan, lewat koneksi WebSocket yang tetap terbuka.

Laravel Reverb adalah server WebSocket resmi dari Laravel (built-in sejak Laravel 11) — alternatif self-hosted untuk Pusher, tanpa biaya layanan pihak ketiga. 

// Tanpa broadcasting (polling — boros request):
Client -> GET /api/produk/5 setiap 3 detik -> Server

// Dengan broadcasting (real-time — server yang push):
Client -> buka koneksi WebSocket sekali -> Server
                                        |
                        [stok produk berubah] -> event di-broadcast
                                        |
Client <- update diterima instan <- Server
Contoh Use Case
Notifikasi "Stok hampir habis" ke dashboard admin secara instan, update status pesanan (diproses → dikirim → selesai) tanpa refresh halaman, atau live chat customer service.

Install & Setup Reverb

Langkah 1 — Install

Terminal
php artisan install:broadcasting

Perintah ini menginstall Reverb, mempublish config, dan menambahkan variabel Reverb ke .env secara otomatis.

Langkah 2 — Konfigurasi .env

.env
BROADCAST_CONNECTION=reverb

REVERB_APP_ID=laravel-crud-app
REVERB_APP_KEY=local-key
REVERB_APP_SECRET=local-secret
REVERB_HOST=127.0.0.1
REVERB_PORT=8080
REVERB_SCHEME=http

Langkah 3 — Jalankan Server Reverb

Terminal (biarkan berjalan, terpisah dari php artisan serve)
php artisan reverb:start
Dua Server Berjalan Bersamaan
php artisan serve (HTTP biasa) dan php artisan reverb:start (WebSocket) adalah dua proses terpisah yang perlu berjalan bersamaan saat development.

Membuat & Broadcast Event

Langkah 1 — Buat Event

Terminal
php artisan make:event StokProdukMenipis
app/Events/StokProdukMenipis.php
<?php

namespace App\Events;

use App\Models\Produk;
use Illuminate\Broadcasting\Channel;
use Illuminate\Broadcasting\InteractsWithSockets;
use Illuminate\Broadcasting\PresenceChannel;
use Illuminate\Contracts\Broadcasting\ShouldBroadcast;
use Illuminate\Foundation\Events\Dispatchable;
use Illuminate\Queue\SerializesModels;

class StokProdukMenipis implements ShouldBroadcast
{
    use Dispatchable, InteractsWithSockets, SerializesModels;

    public function __construct(
        public readonly Produk $produk
    ) {}

    public function broadcastOn(): array
    {
        return [
            new Channel('admin-dashboard'),
        ];
    }

    public function broadcastAs(): string
    {
        return 'stok.menipis';
    }

    public function broadcastWith(): array
    {
        return [
            'produk_id' => $this->produk->id,
            'nama_produk' => $this->produk->nama_produk,
            'stok' => $this->produk->stok,
        ];
    }
}

Langkah 2 — Dispatch dari Service

app/Services/ProdukService.php
use App\Events\StokProdukMenipis;

public function update(Produk $produk, array $data): Produk
{
    $produk = $this->produkRepository->update($produk, $data);

    if ($produk->stok <= 5) {
        StokProdukMenipis::dispatch($produk);   // broadcast otomatis dikirim
    }

    return $produk;
}
Event Biasa vs ShouldBroadcast
Event Laravel biasa hanya berjalan di server (untuk listener internal). Menambahkan interface ShouldBroadcast membuat event yang sama juga dikirim lewat WebSocket ke client yang sedang mendengarkan channel terkait.

Channel & Otorisasi

Jenis ChannelSiapa yang Bisa DengarContoh
PublicSiapa saja, tanpa autentikasiPengumuman umum, harga saham
PrivateUser yang lolos otorisasiprivate-order.{id} — hanya pemilik order
PresencePrivate + tahu siapa saja yang onlineLive chat, siapa yang sedang melihat halaman

Private Channel untuk Update Status Order

routes/channels.php
use App\Models\Order;

Broadcast::channel('order.{orderId}', function ($user, $orderId) {
    // hanya izinkan user yang memiliki order ini
    return $user->id === Order::find($orderId)?->user_id;
});
app/Events/OrderStatusUpdated.php (broadcastOn)
public function broadcastOn(): array
{
    return [
        new PrivateChannel("order.{$this->order->id}"),
    ];
}

Listen di Frontend dengan Laravel Echo

Terminal
npm install --save-dev laravel-echo pusher-js
resources/js/echo.js
import Echo from 'laravel-echo';
import Pusher from 'pusher-js';

window.Pusher = Pusher;

window.Echo = new Echo({
    broadcaster: 'reverb',
    key: import.meta.env.VITE_REVERB_APP_KEY,
    wsHost: import.meta.env.VITE_REVERB_HOST,
    wsPort: import.meta.env.VITE_REVERB_PORT,
    forceTLS: false,
});
Dengarkan Event di JavaScript
// Public channel — dashboard admin
window.Echo.channel('admin-dashboard')
    .listen('.stok.menipis', (e) => {
        console.log(`Stok ${e.nama_produk} tersisa ${e.stok}`);
        // tampilkan notifikasi toast di UI
    });

// Private channel — status order milik user yang login
window.Echo.private(`order.${orderId}`)
    .listen('.order.updated', (e) => {
        console.log(`Status order: ${e.status}`);
    });
Nama Event Diawali Titik
.listen('.stok.menipis', ...) — titik di depan berarti nama event dipakai persis seperti yang dikembalikan broadcastAs(), tanpa prefix namespace class event PHP-nya.
